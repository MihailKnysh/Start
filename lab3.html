Основи Internet

Internet
система взаємопов’язаних комп’ютерних мереж, які використовують стек протоколів TCP\IP для зв’язку пристроїв між собою.

Internet має широкий спектр інформаційних ресурсів та послуг: взаємопов’язані гіпертекстові документи, електронна пошта, IP телефонія та ін.
Ключові організації, пов’язані з інтернет:

    ICANN
    некомерційна організація, яка займається доменними іменами та IP адресами;
    IETF
    відкрите міжнародне товариство проектувальників, учених і провайдерів, які займаються розвитком протоколів та архітектури Internet;
    ISOC
    організація фахівців, яка формує політику і практику Internet; стандартизація Internet. Ця організація володіє правами на документи RFC (документи, які містять технічну документацію і стандарти Internet).
Етапи розвитку мережі Internet:
  • 1961 – ARPANET;
  • 1973 – DNS;
  • 1983 – NFSNET;
  • 1990 – INTERNET.

Internet як мережа

Рівні моделі OSI

  1. Прикладний – HTTP, FTP, Telnet;
  2. Транспортний – TCP, UDP;
  3. Між мережевий – IP;
  4. Канальний – Ethernet.
Для доступу до мережевого ресурсу використовується так званий URL:
URL: schema: [//authority] path [? query] [#fragment]
schema - протокол;
authority - [userinfo@] host [: port];
path - шлях для уточнення вказівки на місце знаходження ресурсу;
query - рядок параметра виду “параметр = значення”, розділених символом `&`.
Приклади
ftp://user:pass@ftp.example.org – authority
http://example.org/script.php?key = value

Web – служба (www)

Web
це інформаційний простір, де документи та інші Web – ресурси, які ідентифікуються URL, пов’язані між собою гіпертекстовими посиланнями та доступними через Internet.
Протокол HTTP
протокол передачі гіпертексту. Версії:
  • HTTP/09 – 1992;
  • HTTP/1 – 1996;
  • HTTP/1.1 – 1999;
  • HTTP/2 – 2015.
    • Основою HTTP є технологія клієнт – сервер. Кожне HTTP повідомлення складається із:
    1. Стартовий рядок;
    2. Заголовки;
    3. Тіло повідомлення.
    Стартовий рядок визначає тип повідомлення. Формат для запиту і відповіді різний. Запит: Метод URL HTTP Версія
    Метод – тип запиту. Існують:
    • GET;
    • PUT;
    • POST;
    • DELETE.
    Приклад
    GET/wiki/HTTP HTTP/1.1

    Відповідь: HTTP Версія Код стану Пояснення
    1хх – інформаційний клас;
    2хх – успіх;
    3хх – перенаправлення;
    4хх – помилка клієнта;
    5хх – помилка сервера.
    Приклади
    404 – Not found
    500 - Internal
